# Security Policy

## Reporting a vulnerability

Do not publish exploitable details in a public issue. Use the repository's
GitHub private vulnerability reporting feature or contact the maintainers
through a private channel listed in the repository profile.

Include the affected version, reproduction steps, expected impact, and any
suggested mitigation. Do not include credentials, audit databases, machine
reports, or other personal data.

## Product boundary

IB Audit Workstation is a read-only local assessment tool. It does not exploit
findings or automatically change Windows configuration. A missing match is not
proof that a system is secure; review coverage, source freshness, confidence,
and insufficient-data statuses.
