# Локальные сторонние инструменты

Каталог `tools/` предназначен только для локальных копий сторонних программ. Все его содержимое, кроме этого файла, исключено из Git.

Исключение из Git не удаляет локальные файлы и не мешает лицензированной локальной сборке. Оно предотвращает случайную публикацию бинарников и не распространяет MIT-лицензию проекта на сторонние компоненты.

## Структура

```text
tools/
|-- README.md
|-- nmap/          # копия Nmap, полученная пользователем на законном основании
|-- wireshark/     # локальный runtime tshark/Wireshark
`-- npcap-oem/     # только Npcap OEM, если лицензия разрешает включение в продукт
```

Обычный `build/pyinstaller/IBAuditWorkstation.spec` создаёт профиль `community` и не включает эти каталоги в EXE.

Для локальной сборки с компонентами используйте:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_licensed_exe.ps1 -IConfirmRights
```

Если действующая Npcap OEM-лицензия разрешает включение драйвера в конкретный продукт:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_licensed_exe.ps1 -IConfirmRights -IncludeNpcapOem
```

## Обязательные условия

- Скрипт ничего не скачивает и использует только локальные файлы пользователя.
- `npcap*.exe`, случайно находящийся внутри `tools/nmap`, всегда исключается из Nmap-архива.
- Бесплатный/demo Npcap нельзя помещать в `tools/npcap-oem`.
- Наличие файлов в `tools/` не подтверждает права на их использование или распространение.
- Лицензия должна покрывать назначение сборки, организацию, продукт и предполагаемый способ распространения.
- Полученный локальный EXE нельзя публиковать только на основании MIT-лицензии этого проекта.
- Для внешнего распространения Wireshark/tshark необходимо выполнить GPL-требования, включая предоставление соответствующего исходного кода.
