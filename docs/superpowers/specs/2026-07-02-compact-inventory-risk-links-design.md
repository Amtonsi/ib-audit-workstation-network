# Компактные ссылки на риски в полном инвентаре

## Цель

Для каждого объекта полного инвентаря, которому сопоставлены подтверждённые
риски, показать компактный список идентификаторов CVE/БДУ. Каждая ссылка должна
переводить пользователя к соответствующей карточке риска того же компьютера.

## Представление

- Список располагается сразу после таблицы параметров объекта.
- Заголовок строки: `Риски:`.
- Каждый риск отображается компактной ссылкой с идентификатором правила,
  например `CVE-2024-0057` или `BDU:2024-01234`.
- Цвет ссылки соответствует критичности риска.
- Ссылки переносятся на следующую строку внутри карточки и не выходят за её
  границы.
- Если с объектом не связан ни один риск, строка не выводится.

## Навигация

Каждая карточка риска компьютера получает устойчивый HTML-якорь, построенный из
якоря компьютера, идентификатора риска и порядкового номера. Ссылка из полного
инвентаря вызывает существующий механизм раскрытия секции компьютера, затем:

1. раскрывает документ выбранного компьютера;
2. прокручивает страницу к конкретной карточке риска;
3. обновляет адресный hash;
4. временно подсвечивает целевую карточку.

Идентификатор цели передаётся в обработчик навигации как отдельный параметр.
Существующие ссылки на общий раздел рисков и полный инвентарь продолжают
работать без изменений.

## Связь данных

Риски связываются с объектами только по `RuleResult.object_uid` и
`InventoryObject.uid`. Эвристическое сопоставление по названию не применяется,
чтобы ссылка не указывала на риск другого экземпляра ПО, драйвера или
устройства.

Если один объект имеет несколько результатов с одинаковым идентификатором,
каждый результат сохраняет собственную целевую карточку, но подписи ссылок
остаются короткими и понятными.

## Безопасность и устойчивость

- Все идентификаторы и подписи экранируются перед добавлением в HTML.
- Якоря нормализуются в допустимую безопасную форму.
- Отсутствующая цель не вызывает ошибку JavaScript: документ раскрывается, а
  прокрутка выполняется к секции рисков компьютера.
- Отчёт остаётся автономным и не загружает внешние скрипты или стили.

## Проверка

Автоматические тесты должны подтверждать:

- наличие компактной строки рисков у объекта с сопоставленным результатом;
- наличие ссылки на конкретную карточку риска;
- уникальность якорей при повторяющихся идентификаторах;
- отсутствие строки рисков у объекта без риска;
- сохранение существующей навигации между рисками и полным инвентарём.

После тестов создаётся демонстрационный HTML-отчёт. В браузере проверяются
компактность строки, перенос длинного списка, раскрытие нужного компьютера,
прокрутка и подсветка целевой карточки.
